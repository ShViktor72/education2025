# Техническое задание: Веб-приложение с использованием облачной базы данных.

## 1. Цель проекта:
Разработать полнофункциональное веб-приложение с использованием **Firebase Firestore** и **Authentication**, реализовав систему пользователей, основной функционал предметной области и админ-панель.

## 2. Выбор темы проекта:
Студент выбирает одну тему из списка:
* **Торговля:** Интернет-магазин, Маркетплейс handmade, Доска объявлений, Сервис доставки еды.
* **Контент и СМИ:** Новостной сайт / Блог, Книга рецептов, Сайт отзывов о фильмах/книгах, Онлайн-библиотека.
* **Бронирование:** Бронирование столиков в ресторане, отелей, билетов в кино, Аренда жилья.
* **Услуги и трекеры:** Система записи к врачу, Тренажер карточек (Anki), Платформа мероприятий, Маркетплейс услуг (фриланс), Фитнес-трекер.

---

## 3. Функциональные требования:

### 3.1 Главная страница (index.html)
* **Каталог:** Вывод основных сущностей с пагинацией (по 10-20 элементов, кнопка «Показать еще»).
* **Поиск:** Реальный поиск по названию/описанию через запросы Firestore.
* **Фильтрация:** По категориям, тегам или параметрам.
* **Сортировка:** По релевантным параметрам (дата, цена, рейтинг и т.д.).
* **Режим реального времени:** Новые элементы появляются без перезагрузки (актуально для обновлений админом).

### 3.2 Страница детального просмотра
* **Инфо:** Полная информация об элементе.
* **Интерактив:** Система взаимодействия (отзывы, комментарии, оценки или форма бронирования).
* **Связи:** Показ связанных или похожих элементов с пагинацией.
* **Действие:** Основная кнопка (Добавить в корзину / Забронировать / Сохранить).
* **Обновление:** Режим реального времени для статусов и доступности.

### 3.3 Личный кабинет
* **История:** Список прошлых действий (заказы, бронирования, публикации).
* **Управление:** Возможность редактировать или удалять свои отзывы и комментарии.
* **Профиль:** Редактирование данных пользователя (имя, контакты, настройки).
* **Статусы:** Обновление статусов текущих действий в реальном времени.

### 3.4 Функциональная страница (cart.html / bookings.html)
* **Менеджмент:** Список активных действий с возможностью управления (удаление, изменение кол-ва).
* **Финализация:** Основное действие (Оформить заказ / Подтвердить / Завершить).
* **Переход:** После завершения данные перемещаются в раздел «История».

### 3.5 Админ-панель (admin.html)
> **Ограничение:** Только для пользователей с ролью `admin`.
* **CRUD:** Управление основными сущностями (добавить, редактировать, удалить).
* **Менеджмент:** Просмотр всех действий пользователей и изменение их статусов.
* **Пользователи:** Список пользователей и назначение прав доступа.
* **Модерация:** Удаление некорректного контента.
* **Статистика:** Базовые метрики проекта (опционально).

### 3.6 Система авторизации
* **Вход:** Регистрация, логин и выход.
* **Безопасность:** Восстановление пароля и сохранение сессии.
* **Роли:** Разграничение прав доступа: `user` (по умолчанию) и `admin`.

## 4. Требования к структуре базы данных:



### 4.1 Обязательные коллекции:
1.  **Пользователи:** Профили, роли и настройки.
2.  **Основные сущности:** Каталог товаров/услуг (зависит от темы).
3.  **Действия:** История заказов, отзывы, лог бронирований.
4.  **Активные действия:** Временные данные (корзина, текущие сессии бронирования).

---

## 5. Технические требования:

### 5.1 Технологии:
* **Firebase Firestore:** Основная база данных.
* **Firebase Authentication:** Система управления пользователями.
* **Firebase Security Rules:** Настройка правил доступа.
* **Frontend:** Vanilla JavaScript (ES6+ модули), HTML5, CSS3 (адаптивная верстка).

### 5.2 Обязательные Firebase-функционалы:
* **onSnapshot:** Использование Real-time минимум в одном модуле.
* **Пагинация:** Реализация минимум в одном списке.
* **Оптимизация:** Настройка индексов, оптимизированные запросы (`select()`), дублирование данных для производительности.
* **Безопасность:** Четкие правила доступа для ролей `user` и `admin`.
