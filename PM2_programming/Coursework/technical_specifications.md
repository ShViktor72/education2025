# Техническое задание: Веб-приложение с использованием JavaScript

## 1. Цель проекта:
Разработать полнофункциональное веб-приложение с использованием JavaScript для закрепления навыков работы с DOM, событиями, массивами объектов и локальным хранилищем. Проект должен представлять собой интерактивный инструмент для учета, планирования или коллекционирования, с понятным интерфейсом и сохранением данных между сессиями.

## 2. Выбор темы проекта:
Студент выбирает одну тему из списка или предлагает свою (согласовав с преподавателем):
1. **Коллекция фильмов/сериалов** — каталог для отслеживания просмотренного.
2. **Список покупок** — интерактивный список с категориями.
3. **Заметки** — личный блокнот для быстрых записей.
4. **Трекер привычек** — отслеживание ежедневных полезных дел.
5. **Планировщик задач (Todo list)** — список дел с приоритетами.
6. **Личный дневник** — записи с оценкой настроения.
7. **Список книг** — библиотека для отслеживания прочитанного.
8. **Трекер расходов** — учет ежедневных трат.
9. **Плейлист музыки** — коллекция любимых треков.
10. **Список подарков** — идейник для близких (кому, что, повод).

## 3. Функциональные требования:

### 3.1 Главная страница (index.html)
Страница отображения всех созданных элементов.
**Обязательные элементы:**
- **Шапка (header)** с навигационным меню (ссылки на все страницы проекта).
- **Заголовок страницы** (например, "Мои фильмы", "Список задач").
- **Список элементов** — каждый элемент отображается в виде карточки или строки таблицы.
- **Карточка элемента** должна содержать:
    - Основную информацию (название, дату, категорию и т.п. в зависимости от темы).
    - Чекбокс или переключатель для изменения статуса (выполнено/не выполнено, куплено/не куплено, просмотрено/не просмотрено).
    - Кнопку "Удалить" (с иконкой или текстом) — при нажатии появляется подтверждение через `confirm`.
- **Кнопка "Добавить"** (ведет на страницу с формой).

### 3.2 Страница с формой (form.html)
Страница для создания нового элемента.
**Обязательные элементы:**
- **Шапка (header)** с навигационным меню (такая же, как на главной).
- **Заголовок страницы** (например, "Добавить новый фильм").
- **Форма** с полями ввода (минимум 3 поля, например: название, категория, дата).
- **Валидация** — проверка, что все обязательные поля заполнены перед отправкой (через JavaScript, не через required в HTML). 
При ошибке — `alert` с сообщением.
- **Кнопка "Сохранить"** — добавляет элемент в общий список и сохраняет в localStorage.
- **Кнопка "Отмена"** или "На главную" — возвращает на главную страницу без сохранения.

### 3.3 Страница "О проекте" (about.html)
Страница с информацией о проекте и авторе.
**Обязательные элементы:**
- **Шапка (header)** с навигационным меню.
- **Название проекта** и его цель (для чего это приложение).
- **Информация об авторе**:
    - Имя и фамилия.
    - Группа/курс.
    - Контакты (email или ссылка на GitHub).
- **Инструкция по использованию** (кратко: как добавлять, удалять, отмечать).
- **Иконки или иллюстрации** (по желанию) для украшения.

### 3.4 Дополнительная страница (по выбору студента)
Студент может добавить страницу на свой выбор, например:
- **stats.html** — страница со статистикой (сколько добавлено за неделю, самые популярные категории).
- **settings.html** — настройки оформления (тема, цвет фона).
- **archive.html** — архив выполненных/удаленных элементов.

## 4. Технические требования:
### 4.1 Технологии:
- **HTML5** — семантическая верстка.
- **CSS3** — адаптивная верстка с использованием Flexbox или Grid.
- **JavaScript (ES6+)** — чистый JS, без фреймворков.
- **LocalStorage** — для хранения данных между сессиями.
- **GitHub Pages** — для публикации готового проекта.

### 4.2 Структура данных:
Все данные хранятся в виде **массива объектов** в localStorage. Каждый объект должен иметь уникальный идентификатор (`id`), который можно создать через `Date.now()` или `crypto.randomUUID()`.

### 4.3 Требования к коду:
- Код должен быть **разделен на модули**, например:
    - `main.js` — логика главной страницы (отрисовка, удаление, изменение статуса).
    - `form.js` — логика страницы с формой (добавление, валидация).
    - `storage.js` — функции для работы с localStorage (get, set).
    - `utils.js` — вспомогательные функции.
- Все функции должны быть **именованными** (не анонимными, кроме стрелочных в колбэках).
- Переменные должны иметь **понятные названия** (на английском языке).
- Код должен быть **отформатирован** (отступы, пробелы).

### 4.4 Требования к интерфейсу:
- **Адаптивность** — приложение должно корректно отображаться на мобильных устройствах (до 320px) и десктопах.
- **Уникальный стиль** — каждый студент придумывает свою цветовую схему:
    - Цвет шапки.
    - Цвет кнопок.
    - Цвет карточек.
    - Цвет фона.
    - Шрифты (можно использовать Google Fonts).
- **Читаемость** — текст должен контрастировать с фоном.
- **Состояния кнопок** — кнопки должны иметь hover-эффекты.

## 6. Этапы реализации:
### Этап 1: Подготовка
- Выбрать тему проекта.
- Создать папку проекта и структуру файлов.
- Нарисовать примерный макет (на бумаге или в Figma).

### Этап 2: Верстка 
- Создать HTML-каркас всех трех страниц.
- Написать базовые CSS-стили (общие для всех страниц).
- Добавить навигационное меню в шапку.
- Сверстать карточку элемента (примерную) для главной страницы.
- Сверстать форму.

### Этап 3: JavaScript — логика данных 
- Создать файл `storage.js` с функциями `getData()` и `saveData()`.
- Создать тестовый массив с 2-3 элементами для отладки.
- Сохранить тестовый массив в localStorage при первой загрузке.

### Этап 4: JavaScript — главная страница 
- Написать функцию `renderItems()`, которая отрисовывает элементы из массива.
- Добавить обработчик на чекбоксы (изменение статуса + перерисовка).
- Добавить обработчик на кнопки удаления (подтверждение + удаление + перерисовка).

### Этап 5: JavaScript — страница с формой 
- Написать функцию валидации (проверка заполнения полей).
- Написать обработчик отправки формы (событие `submit`).
- Создавать новый объект с уникальным id (можно использовать `Date.now()`).
- Добавлять объект в массив и сохранять в localStorage.
- Перенаправлять на главную страницу (через `window.location.href`).

### Этап 6: Страница "О проекте" 
- Заполнить информацию о себе и проекте.
- Добавить стили для этой страницы.

### Этап 7: Тестирование и доработка 
- Проверить все сценарии: добавление, удаление, изменение статуса.
- Проверить, что данные сохраняются после перезагрузки.
- Протестировать на мобильном устройстве (или в режиме эмуляции в браузере).
- Исправить ошибки.

### Этап 8: Публикация 
- Создать репозиторий на GitHub.
- Загрузить код.
- Включить GitHub Pages в настройках репозитория.
- Проверить, что проект работает по опубликованной ссылке.

### Этап 9: Сдача проекта
- Отправить ссылку на репозиторий и на GitHub Pages преподавателю.
